ベクトルで「似ている」を測る

文章を 768 個の数字（ベクトル）に変えると、「似ている」を計算できます。 その数字のどこが似ているのかを取り出し、空間の意味のある向きを見つけるまでを図解します。

Part 1

似ている箇所はどこ？

STEP 1 ・ 直感

「似ている」＝ 2本の矢印の角度が小さいこと

文章を矢印（ベクトル）にすると、似た文章ほど矢印が同じ向きを向きます。 その「向きの近さ」がコサイン類似度です。

cosθ は −1〜1。1 に近いほど同じ向き（似ている）、0 は無関係、−1 は真逆。 内積を長さで割った「正規化内積」が正体です。

STEP 2 ・ 文章 → 768次元ベクトル

埋め込みモデルが文章を 768 個の数字にする

まず文章をトークン（語のかけら）に割り、各トークンを 768 次元のベクトルにします。 文全体のベクトルは、その平均（mean-pooling）です。

# 文Aのベクトル ＝ 全トークンのベクトルを平均する（mean-pooling） A = ( t₁ + t₂ + … + tₙ ) / n

この「平均しているだけ」という事実が、次の STEP の分解を可能にします。

STEP 3 ・ 核心

類似度を「各トークンの寄与」に厳密分解できる

平均は足し算。足し算はバラして考えられるので、 コサイン類似度を1トークンずつの寄与の合計に正確に分けられます。

cos(A,B) = Â · B̂ = ( 寄与₁ + 寄与₂ + … + 寄与ₙ ) ← 総和がぴったり cos に一致 # トークン i の寄与 ＝ そのトークンが「相手の文の向き」へどれだけ効くか 寄与ᵢ = ( tᵢ · B̂ ) / ( n × |A| )
Â,B̂＝長さ1に正規化したベクトル tᵢ＝トークン i のベクトル 「·」＝内積（向きの一致度）
実データで検証済み：桃太郎×浦島太郎で cos = 0.7271、トークン寄与の総和も 0.7271（差は文頭・文末の特殊トークン分のみ ≈ 0.002）。数式は本当に成り立ちます。
STEP 4 ・ 落とし穴と工夫

そのままだと「助詞」ばかり光る → 中心化で内容語を出す

生の寄与を大きい順に並べると、「の・を・と・した」のような機能語が上位を独占します （どの文にも出るので相手の向きと内積が大きくなりがち）。そこで各語の効き具合から 全語の平均を引く（中心化コントラスト）と、内容語が浮き上がります。

Part 2

768次元から意味を取り出す

STEP 1 ・ 直感

768次元は多すぎる ― 情報は少数の方向に偏っている

768 本の軸のうち、ほとんどはほぼ使われていません。 情報が乗っている重要な方向だけを取り出せば、空間の意味が見えます。

実データ：上位 20 本の軸だけで、全体のばらつき（分散）の 52% を説明できます。 768 次元でも、効いているのはごく一部。
STEP 2 ・ PCA

点が最も散らばる向き ＝ 第1主成分（いちばん意味のある軸）

文書を点として置くと、いちばん大きく散らばる向きが見つかります。これが第1主成分。 次に直交する中で最大の向きが第2主成分…と続きます。
