# インフォグラフィック生成プロンプト（技術解説用）

このプロジェクト（謎の国家 RAG）の技術トピックを「**技術書のようにわかりやすい
図解インフォグラフィック**」にするための、再利用可能な指示プロンプト。
新しいトピックを図解したいときは、下の「差し込み」を埋めてこのまま使う。

---

## そのまま使えるプロンプト本文

> 次の技術トピックを、**1枚の自己完結 HTML インフォグラフィック**にしてください。
> 機械学習・NLP の初学者〜中級者が「読めばスッと腑に落ちる」教科書のような図解にすること。
>
> **トピック**：｛ここに対象（例：コサイン類似度のトークン寄与分解）｝
> **使える実データ**：｛実際の数値・例。例：桃太郎×浦島太郎 cos=0.7271｝
>
> ### 各トピックの構成（この順で並べる）
> 1. **一言サマリ** … このトピックで結局何がわかるのか（太字1〜2文）
> 2. **直感** … 身近なたとえ（矢印の角度、影、地図など）。まず絵で理解させる
> 3. **仕組み（ステップ図）** … 入力→…→出力を矢印でつなぐ。各ステップに小さな図
> 4. **核心の数式** … 記号の意味を吹き出し/注釈で必ず添える。「Σ＝全部足す」まで噛み砕く
> 5. **実例** … このプロジェクトの実データで、数式が本当に成り立つことを見せる
> 6. **落とし穴・工夫** … つまずきやすい点と、その回避策
>
> ### デザイン要件（必須）
> - **自己完結**：外部 CDN・画像 URL・フォント読み込みは一切禁止。`<style>`/`<svg>`/インライン JS のみ。
>   オフラインの `file://` でそのまま開けること（この環境はネットワーク隔離のため）。
> - **図は SVG か CSS で自前描画**：ベクトルの矢印＋角度、散布図＋主成分軸、ヒートマップのマス、
>   重心と差ベクトル、などを実際に描く。スクショ画像を貼らない。
> - **教科書トーン**：余白広め、見出し階層、概念ごとに色を固定（例：類似＝赤、相違＝青、軸＝紫）。
>   派手なアニメより**わかりやすさ最優先**。動かすなら「スクロールで1つずつ現れる」程度。
> - **縦1カラム**で上から読むと流れが追えること。各トピックは角丸カードで区切る。
> - 数式は画像でなくテキスト＋SVGで。記号には必ず凡例。
>
> ### 技術的正確性（必須）
> - 数式・用語を正しく（mean-pooling、PCA の説明分散比、コサイン類似度＝正規化内積 など）。
> - 実データの数値は「実際の計算出力」を使い、捏造しない。手元に無ければ計算してから埋める。
>
> ### 出力
> - `docs/` 配下に新規 `.html` を 1 つ Write（この環境では**新規ファイルのみ永続**。既存ファイル編集は不可）。
> - 生成後 `open <file>` でホストのブラウザに表示し、`SendUserFile` でダウンロード用にも渡す。

## このプロジェクトで図解済みのトピック（infographic.html）

1. **コサイン類似度のトークン寄与分解**（似ている箇所ハイライト）
   - 直感：2文 = 2本の矢印、似てる＝角度が小さい＝cosが1に近い
   - 核心：文ベクトル＝トークン平均（mean-pooling）→ 線形だから各トークンの寄与に厳密分解、総和＝cos
   - 工夫：生の寄与は助詞が支配 → 中心化コントラスト（全語平均を引く）で内容語を浮かす
   - 3方式：文アライメント／トークンヒートマップ／トークンペアリング
2. **768次元から意味のある軸を抽出**（次元削減と概念軸）
   - 直感：768次元は多すぎ → 情報が乗っている「重要な方向」だけ見る
   - PCA：点の散らばりが最大になる方向＝第1主成分。実データで上位20軸＝全分散の52%
   - 概念軸：2グループの重心の差ベクトル（例 法令⇄判例）。射影すると1次元で立ち位置が読める

> 補足：トーン調整は「派手より質・グラフは増やさず完成度」がこのユーザーの好み（過去フィードバック）。
