## `python -m rag_system.vector_analysis story`

物語と謎の国家法令のコサイン類似度マトリックス：

```
               桃太郎      浦島太郎     三匹の子豚     シンデレラ   謎の国家・刑法   謎の国家・憲法
     桃太郎     1.000     0.682     0.582     0.443     0.106     0.021
    浦島太郎     0.682     1.000     0.523     0.461     0.100     0.041
   三匹の子豚     0.582     0.523     1.000     0.287     0.150     0.086
   シンデレラ     0.443     0.461     0.287     1.000     0.075     0.037
 謎の国家・刑法     0.106     0.100     0.150     0.075     1.000     0.649
 謎の国家・憲法     0.021     0.041     0.086     0.037     0.649     1.000
```

**読み方:**

- **桃太郎 × 浦島太郎 = 0.682**：旅・主人公・異界の共通モチーフ。
- **桃太郎 × 三匹の子豚 = 0.582**：「童話」というジャンルで近いが内容は別。
- **桃太郎 × 謎の国家・刑法 = 0.106**：ジャンルが完全に違うので無関係。
- **謎の国家・刑法 × 憲法 = 0.649**：同じ法令同士なので高い。

これが Streamlit の **物語ベクトルラボ** のヒートマップで見えているものです。

## `python -m rag_system.vector_analysis benchmark`

```
=== ベクトル操作ベンチマーク ===

1. 埋め込みモデルロード     :   7.49 秒
2. ChromaDB 全件取得 (1354件) :   0.51 秒
3. クエリ埋め込み (1 件)    :   0.40 秒
4. 上位 10 件検索           :   0.00 秒
5. 200x200 類似度行列計算   :   0.00 秒
6. PCA 2D (キャッシュ無し)  :   0.09 秒
7. PCA 2D (キャッシュ有り)  :   0.00 秒
8. t-SNE 2D (キャッシュ無し):   1.51 秒

→ Streamlit の初回ロードは「2 + 6 + 8」の合計が支配的。
  `python -m rag_system.vector_analysis precompute` でキャッシュを作れば
  以降は「2 + 7」（数百ミリ秒）で起動できます。
```

**結論:**

- 埋め込みモデルの初回ロードが最も重い（〜7.5 秒）。
- それ以外はキャッシュさえあれば数百ミリ秒で完結する。
- 1354 件の類似度ランキングは即時（10 ms 未満）。

## `python scripts/vector_space_demo.py`

エンドツーエンドの探索デモ。`scripts/vector_space_demo.py` を直接見ると
コード例として `rag_system.vector_analysis` の使い方が分かります。

抜粋（セクション 3：「童話で書庫を検索する」）:

```
3. 童話をクエリにして謎の国家の書庫を検索する（境界の探究）
クエリ: 桃太郎の物語
本文: 桃から生まれた男の子が鬼ヶ島へ鬼退治に行き、動物を仲間にして宝物を持ち帰った勧善懲悪の物語。…

謎の国家の書庫で最も近い 5 件:
  #1 [類似度 0.313] [民法] legal_framework/civil_code.md
  #2 [類似度 0.283] [判例 (民事)] CIVIL-2022-0385
  #3 [類似度 0.277] [文化規制] legal_framework/cultural_regulations.md
  #4 [類似度 0.277] [憲法] legal_framework/constitution.md
  #5 [類似度 0.274] [刑法] legal_framework/criminal_code.md

→ 童話と最も近い法律チャンクでも類似度はせいぜい 0.3 前後。
  RAG は通常、しっかり遠い文書は引っ張ってこないので安心です。
```
